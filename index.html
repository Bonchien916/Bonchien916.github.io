运维入门第四天-crontab计划任务

发表于 2018-07-06

cron/crontab 计划任务
指定时间点执行任务

语法 crontab 选项
常用选项：
-l （list）列出指定用户的计划任务列表
-e（edit）编辑指定用户的计划任务列表
-u（user）指定的用户名，默认当前用户
-r（remove）删除指定用户的计划任务列表

计划任务以行为单位
分 时 日 月 周 计划任务

例：在0点0分执行reboot
0 0 * reboot

取值范围：
分：0~59
时：0~23
日：1~31
月：1~12
周：0~6

四个符号：

  • *表示取值范围中的每个数字
  • -连续区间，例：表达1~7则用1-7
  • /每多少个，例：每10分钟1次，则在分的位置写*/10
  • ，多个取值，例：在1、2、6点执行，则在时的位置1，2，6

问题与测试

1、每月1、10、22日4：45重启network服务

45 4 1,10,22 service network restart

2、每周六、日的1：10重启network

10 1 6,0 service network restart

3、每天的18:00到23:00间隔半小时重启network服务

*/30 18-23 service network restart

4、每隔两天的上午8点到11点的第三和第十五分钟执行重启

3,15 8-11 /2 * reboot

测试：每1分钟往root家目录中的RT.txt中输入当前时间(追加)

/1 * date +”%F %T” >> /root/RT.txt

contab权限问题

超级管理员可以通过配置文件来设置某些用户不允许设置计划任务
配置文件位置： /etc/cron.deny（黑名单）写用户名，一行一个

还有一个配置文件/etc/cron.allow（白名单）优先级高于黑名单

上海交通大学公开课-数学之旅笔记

发表于 2018-07-04

绪论

通过一个很简单的例子：地球周长加长1M，问一只小老鼠能否钻过去？
这个问题我第一感觉，这怎么可能，地球如此庞大，就算加长1M，实际也没什么变化。其实不然，我们通过计算得到以下答案。

2Π（R+X）=L+1
其中：R为半径
X为半径增量
L为周长
L=2ΠR
∴2ΠR+2ΠX=L+1
∴2ΠX=1
∴X约等于0.16M
通过计算，我们得出半径增量为0.16M，与现实世界中一比照，发现，确实一只小老鼠能够钻过去。

这就是老师建议我们不管是数学中也好，现实生活中也好，遇到问题我们最好能算一算。

七桥问题是18世纪著名古典数学问题之一。在哥尼斯堡的一个公园里，有七座桥将普雷格尔河中两个岛及岛与河岸连接起来。问题是能否从这四块陆地中任何一块出发，恰好通过每一座桥，再回到起点。
1736年欧拉递交了《哥尼斯堡的七座桥》论文，同时开创了数学新的分支——图论与拓扑学。图片来源百度百科。

通过七桥问题引申出数学中抽象的思维，这种思维能够帮助我们在工科学习时更好的解决问题。

积分学

圆面积公式推导：可以将圆分成两个半圆，再将两个半圆分成无数个面积相等的扇形并展开在拼接起来，底边以直线取代
，从而求得面积为周长的一半X原半径，这是一种小学时候的微积分思想。

圆（曲边形）面积的定义是通过黎曼积分定义。

求解圆面积的发展：
欧几里得《几何原本》证明了圆的面积与其直径的平方成正比，a（r）=Ar²，其中A是常数。后来我们都知道了A为Π
刘徽利用正多边形一次次求面积，有点类似极限求面积。
牛顿和莱布尼茨微积分：随着刺激微分学发展的科学问题涌现，比如非匀速运动的瞬间变化问题等。它有一个特点，就是找到积分的一般计算方法。

经过一系列的探索，牛顿莱布尼茨打开了微积分大门。现在把他们的观点作比较：
牛顿从运动学出发，认为它是由位移，通过最初比和最末比导出一个衍生量（速度），所以叫导数，F‘(x)
莱布尼茨从长度不可分量累积成为面积出发，通过除法，得到一个商，所以叫微商，dF/dx
牛顿-莱布尼茨公式 $\int_a^b f(x) {\rm d}x=F(b)-F(a)$
解释：任何一个曲边形在(a,b)区间的面积，算出它的不定积分$F(b)-F(a)$

函数空间

这一节课中讲了距离的定于，引申出范数，又引申出内积空间，最后讲到了拓扑。
距离在数学中的定义：
设X是一非空集合，任给一对这一集合的元素x，y，都给定一个实数d（x,y），与他们对应，并满足：
（1）d（x，y）>=0；d（x，y）=0 <=> x=y
（2）d（x，y）=d（y，x）
（3）d（x，y）<=d（x，z）+d（z，y）
则d（x，y）是这两点间的距离
从距离的定义上可以得到以下观点：一个对象是抽象的，那么就要抓住它最重要的属性。

范数可以定义距离，距离不一定可以定义范数，因为范数的属性更多，相当于热带水果可以定义为水果，但水果不一定是热带水果。

在定义范数前先引入赋范空间的概念：赋予范数的集合。
范数就是赋范空间有向量的模长。
因为范数无夹角，所以引入了内积空间来解决。内积可以导出范数||x||²=（x，x）

内积空间满足：
（1）对称性
（2）对第一变元的线性性
（3）正定性(大于等于0)

最后粗略了解拓扑，它就是弱化了的距离，比距离更加抽象，属性更加少。

运维入门第三天-进阶命令

发表于 2018-07-03

ps 用于查看服务器进程
语法：ps 选项

例：ps -ef | grep xxx
选项：
-e 列出全部
-f 显示全部列（全字段）
字段解释：

UID PID PPID C STIME TTY CMD
执行该进程的用户ID 进程ID 该进程的调用者ID CPU占用率 启动时间 终端设备 该进程名称或路径

top 可以实时动态的查看系统运行情况
语法：top 选项

字段解释：
| PID | USER| PR | VIRT | RES | SHR |S |
| :——:| :——: | :——:| :——: | :——: | :——: |:——: |
| 进程ID | 该进程用户| 优先级 | 虚拟内存 | 常驻内存 | 共享内存 |状态 |
运行时可以使用快捷键
M：将结果按照内存（MEN）从高到低显示
P：CPU使用率从高到低
1：是否展示CPU详细信息

du 查看目录真实大小
语法：du 选项 文件

选项：
-s：summaries只显示汇总大小
-h：高可读性显示

find 查找文件
语法：find 路径范围 选项 选项的值

选项：
-name：按照文档名称模糊搜索
-type：文档类型 “f”表示文件 “d”表示文件夹
例：find /etc -name *.conf 表示搜索etc目录下后缀为conf的文件

service 启动服务
语法：service服务 start/stop/restart

例：service mysql start 启动mysql服务
可以通过ps -ef | grep mysql 来查看服务是否启动成功

kill 杀死进程
语法：kill 进程ID

这样有时候会麻烦，因为首先要知道知道进程ID，所以还有下面一种命令
语法：killall 进程名称

ifconfig 用于操作网卡相关指令

inet addr是网卡的ip地址

reboot 重启linux

shutdown 关机（慎用）
语法：shutdown -h now（root用户使用） 或者shutdown -h 15:30 （具体时间）

那么如何取消关机
在centos7.X之前：ctrl+c
在centos7.X之后：shutdown -c

除了shutdown，关机指令还有init 0 、halt、poweroff

uptime 输出持续在线时间

参数说明：系统当前时间 已运行时间 总连接数 系统平均负载

uname 计算机系统相关信息
语法： uname 选项

例：uname -a （获取完整信息）
参数说明：
类型 主机名 内核版本 发布时间 开源计划

netstat 查看网络连接状态
一般用法：netstat -tnlp

选项说明：
-t：只列出tcp协议的连接
-n：将地址从字母组合转化成ip、协议转化成ip
-l：state状态列中值为LISTEN（监听）的连接
-p：显示对应的PID和进程名称

man 手册
语法：man 命令

运维入门第二天-基本命令(2)

发表于 2018-06-29

df 查看磁盘空间
语法：df -h

free 查看内存使用情况
语法：free -m(以MB为单位查看)

查了下网上的资料，发现也有这样的情况

解释：

total：表示物理内存总量。

used：表示总计分配给缓存（包含buffers 与cache ）使用的数量，但其中可能部分缓存并未实际使用。

free：未被分配的内存。

shared：共享内存。

buffers：系统分配但未被使用的buffers数量。

cached：系统分配但未被使用的cache数量。

buffers/cached中的total、used、free表示如下：

used：实际使用的buffers与cache总量，也是实际使用的内存总量。

free：未被使用的buffers与cache和未被分配的内存之和，这就是系统当前实际可用内存。

total(Mem) = used(Mem) + free(Mem)

total(Mem) = used(buffers/cache) + free(buffers/cache)

used(Mem) = buffers(Mem) + cached(Mem) + used(buffers/cache)

free(buffers/cache) = buffers(Mem) + cached(Mem) + free(Mem)

head/tail 查看文件前/末 n 行（默认10行）
语法：head/tail -n 文件名称

这里tail有第二种语法
tail -f 文件路径查看文件的动态变化
注意：不能手动打开

wc 统计文件内容信息
语法：wc -lwc 需要统计的文件内容

-l：lines行数
-w：words单词数（依据空格来判断数量）
-c：bytes字节数

date 查看日期 CST表示中国
语法：date 选项

例： date +%F(等价于+%Y-%m-%d)

还可以查看之前或者之后的时间。比如这样的语法date -d “-1 day” “+%F”,表示前一天

cal 查看日历
语法：cal 选项


cal -y 2018输出2018年的日历

（重要）管道符：| 通常用来“过滤”、”扩展”、”特殊”

管道前面的输出就是管道后面的输入

grep：主要用于过滤

运维入门第一天-基本命令

1、认识Linux基本目录

bin 二进制文件（几乎所有用户命令），皆可运行
boot Linux启动时用到的一些文件
dev 外接设备，需要挂载使用，常用的是挂载光驱mount /dev/cdrom /mnt
lib 系统使用的函数库，比较重要的目录为/lib/modules
lost+found 系统异常产生错误时，会将一些遗失的片段放置于此目录下
mnt 光盘默认挂载点
srv 服务启动之后需要访问的数据目录，如www服务需要访问的网页数据存放在/srv/www内
etc 配置文件
home 除了root用户以外的其他用户的home目录
proc Linux运行时的进程
root root用户的Home
sbin super二进制文件，可运行，前提是有super权限的用户
temp 系统运行时产生的临时文件，用完会自动清理
usr 用户自己安装的软件
var 放置系统执行过程中经常变化的文件，如随时更改的程序/系统的日志文件/var/log，/var/log/message

2、Linux基本命令

Linux通用格式
指令主体 【选项】 【操作对象】
主体只有一个，选项可以多个，操作对象可以多个

指令
ls(list) 当前目录下所有文件、文件夹的名称

ls 路径 指定路径下的所有文件、文件夹的名称
相对路径：相对要有一个参照物（一般时当前目录下）
绝对路径：根目录开始

语法：ls 选项 路径
ls -l /root
ls -la /root
ls -lha /root
-l : 表示详细显示
-a : 显示所有文件/文件夹，包括隐藏的
-h : 高可读性


pwd 打印当前路径的绝对路径

cd 切换当前目录

语法：cd 路径(绝对路径)
cd /usr/local

常用：cd .. 回到上级目录
cd / 回到根目录
cd ~ 当前用户的home目录

mkdir 创建目录
语法一：mkdir 路径（可以是绝对、相对）
语法二：mkdir -p 绝对路径（此语法可用来一次性创建多层不存在的目录）
当前目录下直接mkdir xxx文件夹，像创建非当前目录的需要mkdir /../../xxx(绝对路径)，然后可以通过ls命令来验证是否创建了这个文件夹，简单不多说。
注意 颜色说明（用的Xshell）：
白色：表示普通文件
蓝色：表示目录
绿色：表示可执行文件
红色：表示压缩文件
浅蓝色：链接文件
红色闪烁：表示链接的文件有问题
黄色：表示设备文件
灰色：表示其他文件

touch 创建文件
语法：touch 绝对路径/文件名称（和mkdir类似）

cp 复制文件/文件夹
语法：cd 被复制的文件的绝对路径/文件名称.后缀 被复制绝对路径
文件夹复制时需要加选项 -r(递归复制)
例：
cd -r ./xxxx/ /home/xxxx

mv 移动、剪切
语法：mv 需要移动文档路径 需要保存的位置
注：重命名也是mv命令

rm 删除
语法：rm 选项 需要删除的文档路径
rm -f xxx 表示取消是否删除，直接删除
如果想删除 整个文件夹包括其中的文件 rm -rf xxx
有时候用到 通配符 ，比如 rm -f xxx**表示删除xxx开头的文件

vim 文本编辑
语法：vim 文件路径
编辑时v进入编辑，需要保存并退出:wq 直接退出:q

cat 直接打开一个文件、合并文件
语法一：cat 文件路径
与vim的区别就是 不需要:q退出
语法二：cat 待合并文件路径1 待合并文件路径2 >合并之后的文件路径
注：这里的>表示输出重定向，意思是只合并，不做后续操作，如果有后续操作用>>
