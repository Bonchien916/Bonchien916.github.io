运维入门第四天-crontab计划任务

发表于 2018-07-06

cron/crontab 计划任务
指定时间点执行任务

语法 crontab 选项
常用选项：
-l （list）列出指定用户的计划任务列表
-e（edit）编辑指定用户的计划任务列表
-u（user）指定的用户名，默认当前用户
-r（remove）删除指定用户的计划任务列表

计划任务以行为单位
分 时 日 月 周 计划任务

例：在0点0分执行reboot
0 0 * reboot

取值范围：
分：0~59
时：0~23
日：1~31
月：1~12
周：0~6

四个符号：

  • *表示取值范围中的每个数字
  • -连续区间，例：表达1~7则用1-7
  • /每多少个，例：每10分钟1次，则在分的位置写*/10
  • ，多个取值，例：在1、2、6点执行，则在时的位置1，2，6

问题与测试

1、每月1、10、22日4：45重启network服务

45 4 1,10,22 service network restart

2、每周六、日的1：10重启network

10 1 6,0 service network restart

3、每天的18:00到23:00间隔半小时重启network服务

*/30 18-23 service network restart

4、每隔两天的上午8点到11点的第三和第十五分钟执行重启

3,15 8-11 /2 * reboot

测试：每1分钟往root家目录中的RT.txt中输入当前时间(追加)

/1 * date +”%F %T” >> /root/RT.txt

contab权限问题

超级管理员可以通过配置文件来设置某些用户不允许设置计划任务
配置文件位置： /etc/cron.deny（黑名单）写用户名，一行一个

还有一个配置文件/etc/cron.allow（白名单）优先级高于黑名单
